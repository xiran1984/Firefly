# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Tech Stack

- **Astro 6.3** — static site generator with content collections, SSR endpoints (`src/pages/api/`, `src/pages/og/`, `src/pages/rss.xml.ts`)
- **Svelte 5** — interactive client components (search, display settings, light/dark switch, share poster, archive panel)
- **Tailwind CSS v4** — utility-first CSS with `@theme` and `@utility` directives in `src/styles/main.css`
- **Stylus** — CSS variables defined in `src/styles/variables.styl` (theme colors, spacing, shadows, etc.)
- **TypeScript** — strict mode, path aliases via `tsconfig.json` (`@/` → `src/`, `@components/` → `src/components/`, etc.)
- **Swup** — page transition animations between routes
- **Pagefind** — client-side full-text search, runs at build time
- **Biome** — formatting and linting (no ESLint/Prettier)

## Commands

```bash
bun run dev          # Start dev server (port 4321)
bun run build        # Full production build: generate-icons → astro build → pagefind
bun run check        # Type-check with astro check
bun run preview      # Preview production build
bun run format       # Format src/ with Biome
bun run lint         # Lint and auto-fix src/ with Biome
bun run new-post     # Scaffold a new blog post via scripts/new-post.js
bun run icons        # Regenerate icon sprite from scripts/generate-icons.js
```

**Important:** The project specifies `pnpm@9.14.4` as its package manager. When pnpm is unavailable, `bun install --ignore-scripts` works as a fallback (the `preinstall` hook runs `npx only-allow pnpm`).

## Architecture

### Layout hierarchy

```
Layout.astro                    # Base: <html>, <head>, OG meta, fonts, analytics, music, global CSS
└── MainGridLayout.astro        # Main content: navbar, sidebars, wallpaper/banner, footer, floating controls
    └── [Page Content]          # Injected via <slot>
```

`Layout.astro` handles global concerns (CSS variables via `--hue`, favicons, analytics, fonts, music player). `MainGridLayout.astro` handles per-page layout: wallpaper rendering, sidebar placement (left/right/both), navbar transparency, floating TOC, and content grid via CSS classes generated by `src/utils/responsive-utils.ts`.

### Configuration system

All config lives under `src/config/` and re-exports from `src/config/index.ts`. Key files:

| File | Purpose |
|------|---------|
| `siteConfig.ts` | Site title, URL, theme color hue, page width, card style, pagination, analytics, image optimization, font, language |
| `backgroundWallpaper.ts` | Wallpaper mode (`banner`/`fullscreen`/`overlay`/`none`), image sources, carousel, navbar transparency, waves animation, gradient |
| `sidebarConfig.ts` | Sidebar layout mode (left/right/both), widget assignments per sidebar |
| `navBarConfig.ts` | Navbar menu items, search config |
| `musicConfig.ts` | Music player playlist and behavior |
| `commentConfig.ts` | Comment system (Giscus/Waline/Twikoo/Artalk/Disqus) |
| `profileConfig.ts` | Author info for profile widgets |
| `fontConfig.ts` | Custom font definitions |
| `expressiveCodeConfig.ts` | Code block theme and plugin settings |

**Pattern:** Most feature toggles live in config, not in component props. Config changes take effect on next dev server restart (HMR picks up most but not all).

### Content

Blog posts are Markdown/MDX files in `src/content/posts/`. The content collection schema is defined in `src/content.config.ts` with frontmatter fields: `title`, `published`, `updated`, `draft`, `description`, `image`, `tags`, `category`, `lang`, `pinned`, `password` (for encrypted posts), `licenseName`, `licenseUrl`.

Static pages (about, friends, guestbook) are in `src/content/spec/`.

`src/utils/content-utils.ts` provides `getSortedPosts()`, `getRelatedPosts()`, tag/category aggregation, and pinned-post ordering. Draft posts are excluded in production builds.

### i18n

Language keys are an enum in `src/i18n/i18nKey.ts`. Translations live in `src/i18n/languages/{zh_CN,zh_TW,en,ja,ru}.ts`. The `i18n()` function in `src/i18n/translation.ts` resolves keys against `siteConfig.lang`, falling back to zh_CN then English. Usage in Astro components: `{i18n(I18nKey.someKey)}`.

### Pages and routing

- `src/pages/posts/[...slug].astro` — blog post detail page with content rendering, comments, related posts, license, share poster
- `src/pages/[...page].astro` — paginated post listing (index page)
- `src/pages/archive.astro` — archive page with full post list
- `src/pages/search.astro` — Pagefind-powered search
- `src/pages/bangumi.astro`, `friends.astro`, `guestbook.astro`, `sponsor.astro`, `gallery/` — feature pages
- `src/pages/api/allPostMeta.json.ts` — API endpoint exposing post metadata
- `src/pages/og/[...slug].png.ts` — OG image generation (disabled by default, slow)
- `src/pages/rss.xml.ts` — RSS feed generation

### Component patterns

- **Astro components** (`*.astro`): Static/server-rendered content. Found in `src/components/layout/`, `common/`, `features/`, `widget/`, `analytics/`, `comment/`, `controls/`, `misc/`, `pages/`.
- **Svelte components** (`*.svelte`): Interactive client-side widgets. Found alongside Astro components in `src/components/`. Key ones: `DisplaySettingsIntegrated.svelte`, `Search.svelte`, `LightDarkSwitch.svelte`, `SharePoster.svelte`, `ArchivePanel.svelte`.

### Wallpaper/background system

Four modes defined in `src/constants/constants.ts`: `banner`, `fullscreen`, `overlay`, `none`. The mode is set in `backgroundWallpaper.ts`. Images can be local (`src/assets/images/` for optimized, `public/assets/images/` for raw) or remote URLs. Desktop and mobile images are configured separately. `src/utils/layout-utils.ts` handles image resolution, and `src/utils/setting-utils.ts` manages runtime wallpaper state (mode switching, overlay opacity, waves animation, gradient).

### Path aliases

Defined in `tsconfig.json`:
- `@/` → `src/`
- `@components/` → `src/components/`
- `@constants/` → `src/constants/`
- `@utils/` → `src/utils/`
- `@i18n/` → `src/i18n/`
- `@layouts/` → `src/layouts/`
- `@images/` → `src/assets/images/`

### Key utilities

- `src/utils/layout-utils.ts` — background image resolution, homepage detection
- `src/utils/content-utils.ts` — post sorting, related posts, tag/category aggregation
- `src/utils/setting-utils.ts` — runtime theme/wallpaper/overlay state management (large file)
- `src/utils/url-utils.ts` — URL generation, slug handling, path parsing
- `src/utils/image-utils.ts` — quality selection, responsive image helpers
- `src/utils/responsive-utils.ts` — CSS class generation for responsive grid/sidebar layouts
- `src/utils/date-utils.ts` — date formatting with dayjs
